# 五子棋游戏设计文档

## 1. 项目概述

### 1.1 项目目标
创建一个智能五子棋游戏，让玩家与能够进行策略性落子并准确判断胜负的算法对战。

### 1.2 核心需求
- **平台**: Web浏览器（HTML/CSS/JavaScript）
- **游戏模式**: 仅人机对战（玩家 vs AI）
- **棋盘**: 标准15x15棋盘
- **AI难度**: 多种难度级别（简单/中等/困难，用户可选择）
- **界面风格**: 中国传统风格（水墨画、书法元素）
- **悔棋功能**: 有限悔棋（只能撤销最近一步）
- **数据持久化**: 不保存游戏记录（关闭后丢失）

## 2. 技术方案

### 2.1 技术栈
- **前端框架**: TypeScript 5.x
- **绘图技术**: HTML5 Canvas
- **构建工具**: 现代前端构建工具（Vite/Webpack）
- **测试框架**: Jest + Cypress

### 2.2 选择理由
- **TypeScript**: 提供类型安全，减少运行时错误，提高代码可维护性
- **Canvas**: 适合绘制中国传统风格的水墨画效果，性能良好
- **模块化架构**: 便于开发、测试和维护
- **纯前端实现**: 无需后端服务，易于部署

## 3. 系统架构

### 3.1 整体架构
游戏采用模块化架构，主要包含五个核心模块：

1. **游戏控制器（GameController）**: 协调所有模块，管理游戏生命周期
2. **棋盘渲染器（BoardRenderer）**: 负责Canvas绘制，实现水墨风格视觉效果
3. **AI引擎（AIEngine）**: 实现Minimax算法与Alpha-Beta剪枝，支持三种难度级别
4. **输入处理器（InputHandler）**: 处理用户点击、悔棋等交互
5. **状态管理器（StateManager）**: 维护游戏状态（当前玩家、棋盘数据、游戏历史）

### 3.2 模块间通信
- 控制器通过事件系统与各模块通信
- 渲染器订阅状态变化并更新Canvas
- AI引擎接收棋盘状态，返回最佳落子位置
- 输入处理器将用户操作转换为游戏事件

## 4. 核心组件设计

### 4.1 游戏控制器（GameController）
**职责**: 管理游戏生命周期，协调各模块

**主要方法**:
- `startGame(difficulty)`: 初始化游戏，设置AI难度
- `makeMove(row, col)`: 处理玩家落子
- `undoMove()`: 撤销最近一步（有限悔棋）
- `checkWin(row, col)`: 检查是否获胜
- `switchPlayer()`: 切换当前玩家

**事件系统**: 触发`gameStart`, `moveMade`, `gameOver`等事件

### 4.2 棋盘渲染器（BoardRenderer）
**职责**: Canvas绘制，实现中国传统风格

**主要方法**:
- `drawBoard()`: 绘制15x15网格，水墨风格线条
- `drawPiece(row, col, player)`: 绘制棋子（黑白水墨效果）
- `draw水墨效果()`: 添加墨迹飞溅、宣纸纹理
- `animateMove(row, col, player)`: 落子动画

**视觉元素**: 宣纸背景、毛笔线条、印章式棋子

### 4.3 AI引擎（AIEngine）
**职责**: 计算最佳落子位置

**主要方法**:
- `getBestMove(board, difficulty)`: 返回最佳位置
- `evaluateBoard(board)`: 棋盘评估函数
- `minimax(board, depth, alpha, beta, maximizing)`: Minimax算法

**难度级别**:
- **简单**: 搜索深度2，随机性较高
- **中等**: 搜索深度4，平衡策略
- **困难**: 搜索深度6，专注进攻和防守

### 4.4 输入处理器（InputHandler）
**职责**: 处理用户交互

**主要方法**:
- `handleClick(event)`: 监听Canvas点击
- `getGridPosition(x, y)`: 将像素坐标转换为网格位置
- `isValidMove(row, col)`: 验证落子合法性

**支持**: 鼠标点击、触摸事件（移动端兼容）

### 4.5 状态管理器（StateManager）
**职责**: 维护游戏状态

**主要属性**:
- `board: number[][]`: 15x15棋盘数组（0空，1黑，2白）
- `currentPlayer: number`: 当前玩家（1或2）
- `moveHistory: Move[]`: 落子历史（支持悔棋）
- `gameStatus: string`: 游戏状态（进行中、黑胜、白胜、平局）

**主要方法**:
- `getBoard()`, `getCurrentPlayer()`, `getMoveHistory()`
- `updateBoard(row, col, player)`, `removeLastMove()`

## 5. 数据流设计

### 5.1 核心数据流

#### 5.1.1 玩家落子流程
1. 用户点击Canvas → InputHandler获取坐标 → 转换为网格位置
2. InputHandler验证落子合法性 → 触发`playerMove`事件
3. GameController接收事件 → 调用StateManager.updateBoard()
4. StateManager更新棋盘数组 → 触发`stateChanged`事件
5. BoardRenderer监听事件 → 调用drawPiece()绘制棋子
6. GameController.checkWin()检查胜负 → 如无胜负切换玩家

#### 5.1.2 AI计算流程
1. GameController切换到AI玩家 → 触发`aiTurn`事件
2. AIEngine.getBestMove()接收当前棋盘状态
3. 执行Minimax算法（深度根据难度调整）
4. 返回最佳落子位置 → 触发`aiMove`事件
5. GameController接收AI落子 → 调用StateManager.updateBoard()
6. 状态更新 → 渲染更新 → 检查胜负

#### 5.1.3 悔棋流程
1. 用户点击悔棋按钮 → InputHandler触发`undoRequest`事件
2. GameController验证悔棋合法性（仅限玩家回合，且有历史记录）
3. 调用StateManager.removeLastMove() → 移除最近两步（玩家+AI）
4. 状态更新 → 触发`stateChanged`事件
5. BoardRenderer重绘整个棋盘（清除最近两步棋子）

### 5.2 状态数据结构
```typescript
interface GameState {
  board: number[][];          // 15x15棋盘，0空，1黑，2白
  currentPlayer: 1 | 2;       // 1黑（玩家），2白（AI）
  moveHistory: Move[];        // 落子历史
  gameStatus: 'playing' | 'black_wins' | 'white_wins' | 'draw';
  difficulty: 'easy' | 'medium' | 'hard';
}

interface Move {
  row: number;
  col: number;
  player: 1 | 2;
  timestamp: number;
}
```

### 5.3 事件系统
- `gameStart(difficulty)`: 游戏开始
- `playerMove(row, col)`: 玩家落子
- `aiMove(row, col)`: AI落子
- `moveMade(move)`: 落子完成（通用）
- `undoMove()`: 悔棋请求
- `gameOver(winner)`: 游戏结束
- `stateChanged(newState)`: 状态变化

## 6. 错误处理设计

### 6.1 用户输入错误
- **无效落子位置**: 点击已有棋子或超出棋盘范围
  - 处理: InputHandler.isValidMove()返回false，显示视觉提示（红色闪烁）
  - 用户提示: 无侵入式提示，不使用alert弹窗
- **非玩家回合点击**: AI思考时玩家点击
  - 处理: GameController忽略点击，显示"AI思考中..."提示

### 6.2 AI计算错误
- **计算超时**: AI算法执行时间过长（超过5秒）
  - 处理: 设置超时机制，返回随机合法位置
  - 用户提示: 控制台警告，不干扰游戏
- **无合法落子位置**: 棋盘已满或异常状态
  - 处理: AIEngine返回null，游戏判定为平局
  - 用户提示: 显示"平局"结果

### 6.3 渲染错误
- **Canvas绘制失败**: 浏览器不支持Canvas或内存不足
  - 处理: 检测Canvas支持，显示降级文本界面
  - 用户提示: 友好提示"您的浏览器不支持Canvas，请升级浏览器"
- **动画卡顿**: 低性能设备动画掉帧
  - 处理: 提供"简化动画"选项，减少特效
  - 用户提示: 设置中可调整动画质量

### 6.4 状态一致性错误
- **状态与渲染不同步**: 内存数据与界面显示不一致
  - 处理: StateManager每次更新后触发完整重绘
  - 用户提示: 自动修复，无需用户干预
- **悔棋状态异常**: 悔棋历史记录损坏
  - 处理: 验证moveHistory完整性，异常时禁用悔棋
  - 用户提示: 显示"悔棋功能暂时不可用"

### 6.5 浏览器兼容性问题
- **旧版浏览器**: 不支持ES6+或TypeScript特性
  - 处理: 使用Babel转译，提供ES5兼容版本
  - 用户提示: 检测浏览器版本，提示升级
- **移动端触摸问题**: 触摸事件与点击事件冲突
  - 处理: 统一使用Pointer Events API
  - 用户提示: 无特殊提示，自动适配

### 6.6 性能问题
- **内存泄漏**: 长时间游戏内存增长
  - 处理: 定期清理事件监听器，重用Canvas上下文
  - 用户提示: 无特殊提示，后台自动管理
- **CPU占用过高**: AI计算导致界面卡顿
  - 处理: 使用Web Worker运行AI算法，避免阻塞主线程
  - 用户提示: 显示"AI计算中..."进度指示
